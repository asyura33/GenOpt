Installation

To run GenOpt, you need to install a Java interpreter, which may be obtained from Sun Microsystems. Sun Microsystems has also Java for Microsoft Windows.

Running GenOpt

Running GenOpt from the file explorer

The easiest way to run GenOpt is to double-click on the file genopt.jar. This will start the graphical user interface.

Running GenOpt from the command line

GenOpt can also be run as a console application, either with or without the graphical user interface. To run GenOpt as a console application with graphical user interface, open a shell, go to this directory and type 

  java -jar genopt.jar [initializatonFile]
where [initializatonFile] is an optional argument that can be replaced with the initialization file (see example below). To start GenOpt without the graphical user interface, type 
  java -classpath genopt.jar genopt.GenOpt [initializatonFile]

For example, to run the minimization of the quadratic function that is provided with the example files using the Hooke-Jeeves algorithm, you can type on Linux and Mac OS X

  java -jar genopt.jar example/quad/GPSHookeJeeves/optLinux.ini
and on Windows 
  java -jar genopt.jar example\quad\GPSHookeJeeves\optWinXP.ini

For further information and how to use your simulation software to evaluate the cost function, please read the manual.

Directories

example

cfg

This directory contains files that show how to configure different simulation programs on various operating systems. The files in this directory contain comments that explain for which program and operating system they were written.

This directory contains examples that use EnergyPlus to evaluate the cost function, and examples that minimizes a quadratic cost function. For illustrating the use of GenOpt, the directory example/quad contains several examples that either minimize a simple quadratic cost function or use the quadratic function to do parametric runs. These examples are recommended as a starting point to learn how GenOpt works.

genopt

The genopt directory contains the source and binary code of GenOpt. The source code of the optimization algorithms can be found in the directory genopt/algorithm. You do not have to compile these files. They are only of interest if you want to implement your own optimization algorithms.

jdoc

This directory contains the code documentation of the package genopt.algorithm. Please refer to it if you implement your own optimization algorithms.
