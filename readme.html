Installation

To run GenOpt, you need to install a Java interpreter, which may be obtained from Sun Microsystems. Sun Microsystems has also Java for Microsoft Windows.

Please refer to the GenOpt manual for how to install and run GenOpt. The manual can be downloaded from http://simulationresearch.lbl.gov/GO.

Directories

genopt

The genopt directory contains the binary code of GenOpt. The directory genopt/algorithm contains also the source code of the optimization algorithms. You do not have to compile these files. They are only of interest if you want to implement your own optimization algorithms.

jdoc

This directory contains the code documentation of the package genopt.algorithm. Please refer to it if you implement own optimization algorithms.

example

There are examples that use EnergyPlus to evaluate the cost function, and examples that minimizes a quadratic cost function. For illustrating the use of GenOpt, the directory example/quad contains several examples that either minimize a simple quadratic cost function or use the quadratic function to do parametric runs. These examples are recommended as a starting point to learn how GenOpt works.

Files

cfg/*.cfg

The cfg files show how to configure different simulation programs on various operating systems.

Please see the notes in the files for which program and operating system they were written.

*.class

The class files that contain the binary code. They will be loaded during the execution of GenOpt.

*.java

The java files contain the source code.

Running GenOpt

To run an example, make sure you set the environment variables as described in the GenOpt manual that can be downloaded from the web.

Alternatively, on Mac OS X or Linux you can open a shell, go to this directory and type 

  source setenv.sh
to set the environment variables. This can also be done on Windows if you have Cygwin installed.

To start GenOpt's graphical user interface, double-click the file genopt.jar. To start GenOpt from a command line, open a shell (on Mac OS X or Linux) or a MS-DOS window (on Windows), and type either

  java genopt.WinGenOpt
or 
  java genopt.GenOpt
The second version will start GenOpt without the graphical user interface.

For example, to run the minimization of the quadratic function using the Hooke-Jeeves algorithm, you can type on Linux and Mac OS X

  cd example/quad/GPSHookeJeeves
  java genopt.WinGenOpt
and on Windows 
  cd example\quad\GPSHookeJeeves
  java genopt.WinGenOpt optXXX.ini
where XXX is the name of your operating system.

Alternatively, you can run GenOpt without the graphical user interface by typing

  java genopt.GenOpt optXXX.ini

For further information and how to use your simulation software to evaluate the cost function, please read the manual.
